2. SEQUENCE DIAGRAM

Create file:

diagrams/sequence_diagram.txt

User → API Gateway → Editor Service → Database
User → Run Agent → API Gateway → Agent Manager
Agent Manager → Message Queue → Agent Worker
Agent Worker → Database → Sync Service
Sync Service → WebSocket → Editor → User

Step Flow

1. User edits file
2. File saved to DB
3. User requests agent
4. Agent Manager creates task
5. Task sent to Queue
6. Agent picks task
7. Agent processes
8. Result saved
9. Sync service updates editor

3. AGENT COMMUNICATION PROTOCOL

Create file:

agents/agent_protocol.md

Agent Communication Protocol

Agents communicate using Message Queue.

Task Message Format:
{
  task_id: int,
  agent_type: string,
  project_id: int,
  file_id: int,
  input_data: string,
  priority: int,
  created_at: timestamp
}

Agent Response Format:
{
  task_id: int,
  status: completed,
  result: string,
  completed_at: timestamp
}

Create file:

architecture/event_driven.md

System uses Event Driven Architecture.

Events:
- UserCreated
- ProjectCreated
- FileUpdated
- AgentTaskCreated
- AgentTaskCompleted
- VersionCreated
- SyncUpdate

Event Flow:
User → Event → Queue → Service → Database → Sync → User

Event Example

FileUpdated Event:
{
  event: FileUpdated,
  project_id: 12,
  file_id: 55,
  updated_by: user_21,
  timestamp: 123456789
}
